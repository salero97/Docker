=========================================
ПОЛЕЗНЫЕ DOCKER КОМАНДЫ (ЗАДАНИЕ 14)
=========================================

1. ПРОСМОТР ЛОГОВ
-----------------
docker logs <контейнер>      - просмотр вывода контейнера
docker logs -f <контейнер>   - потоковый вывод логов (аналог tail -f)

2. ВЫПОЛНЕНИЕ КОМАНД В КОНТЕЙНЕРЕ
---------------------------------
docker exec -ti <контейнер> /bin/bash  - вход в оболочку контейнера
docker exec <контейнер> <команда>      - выполнить команду в контейнере

3. ДЕТАЛЬНАЯ ИНФОРМАЦИЯ
----------------------
docker inspect <контейнер>                 - полная информация в JSON
docker inspect -f '{{.NetworkSettings.IPAddress}}' <контейнер> - только IP

4. ОСТАНОВКА КОНТЕЙНЕРОВ
-----------------------
docker stop <контейнер>            - остановить один контейнер (SIGTERM)
docker stop $(docker ps -q)        - остановить ВСЕ работающие контейнеры

5. УДАЛЕНИЕ КОНТЕЙНЕРОВ
----------------------
docker rm <контейнер>              - удалить один контейнер
docker rm $(docker ps -a -q)       - удалить ВСЕ остановленные контейнеры
docker rm -f $(docker ps -a -q)    - принудительно удалить все контейнеры

6. УДАЛЕНИЕ ОБРАЗОВ
------------------
docker rmi <образ>                 - удалить один образ
docker rmi $(docker images -q)     - удалить ВСЕ образы
docker rmi -f $(docker images -aq) - принудительно удалить все образы

7. ПОЛНАЯ ОЧИСТКА СИСТЕМЫ
------------------------
docker system prune               - удалить неиспользуемые объекты
docker system prune -a            - удалить ВСЕ неиспользуемые объекты
docker system prune -a --volumes  - удалить всё + тома (самая агрессивная)

8. ПРОВЕРКА РЕСУРСОВ
-------------------
docker system df                  - использование диска
docker ps                         - работающие контейнеры
docker ps -a                      - все контейнеры (включая остановленные)
docker images                     - все образы

9. ФИЛЬТРАЦИЯ
-------------
docker ps -a --filter name=<имя>  - фильтр по имени
docker rm $(docker ps -a -q --filter name=<имя>) - удалить по фильтру

=========================================
КЛЮЧЕВЫЕ ФЛАГИ:
=========================================
-ti    : интерактивный режим с псевдотерминалом
-d     : фоновый режим (демон)
--rm   : автоматически удалить контейнер после остановки
--name : задать имя контейнеру
-v     : подключить том (volume)
-p     : проброс портов (host:container)
-f     : принудительно (force)
-a     : все (all)
-q     : только ID (quiet)

=========================================
ВАЖНО:
=========================================
1. Команды с $(...) выполняют подстановку
2. docker system prune -a --volumes удалит ВСЁ неиспользуемое
3. Перед удалением образов нужно удалить контейнеры
4. Флаг --rm автоматически удаляет контейнер после stop
